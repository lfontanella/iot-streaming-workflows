# Arduino IDE / Arduino IoT Cloud Prototype

This folder documents the first prototype developed for the acquisition and visualization of environmental data using an Arduino MKR WiFi 1010 board and a DHT22 temperature and humidity sensor.

The objective of this prototype is to test a basic sensor-to-dashboard workflow before moving toward MQTT-based, proprietary, open-source and BIM/openBIM visualization workflows.

## Prototype overview

The prototype is based on the following workflow:

```text
DHT22 temperature and humidity sensor
        ↓
Arduino MKR WiFi 1010
        ↓
Arduino IoT Cloud
        ↓
Real-time 2D dashboard
```

The sensor collects temperature and relative humidity values. These values are transmitted to Arduino IoT Cloud and displayed through a simple real-time dashboard.

## Hardware

The prototype uses:

- Arduino MKR WiFi 1010;
- DHT22 temperature and humidity sensor;
- jumper wires;
- Wi-Fi connection;
- Arduino IoT Cloud account.

## Wiring

The wiring diagram is included in the `images` folder.

In the tested configuration:

- the DHT22 data pin is connected to digital pin 7;
- the sensor is powered through the Arduino board;
- the ground pin is connected to GND.

## Arduino sketch

The Arduino sketch is included in the `TempHumidity` folder.

Main files:

| File | Description |
|---|---|
| `TempHumidity.ino` | Main Arduino sketch for reading temperature and humidity from the DHT22 sensor. |
| `thingProperties.h` | Arduino IoT Cloud properties generated for the temperature and humidity variables. |
| `arduino_secrets.example.h` | Example file for Wi-Fi credentials. Real credentials must not be committed to the repository. |
| `sketch.json` | Arduino Cloud sketch configuration file. |
| `ReadMe.adoc` | Original Arduino sketch documentation file generated by Arduino Cloud. |

## Arduino IoT Cloud variables

The prototype uses two cloud variables:

| Variable | Type | Description |
|---|---|---|
| `temperature` | `float` | Temperature value read from the DHT22 sensor. |
| `humidity` | `float` | Relative humidity value read from the DHT22 sensor. |

These variables are visualized in Arduino IoT Cloud through dashboard widgets.

## Dashboard

## How to use the sketch

To reproduce the prototype:

1. create an Arduino IoT Cloud Thing;
2. define two cloud variables: `temperature` and `humidity`;
3. connect the DHT22 sensor to the Arduino MKR WiFi 1010 board;
4. upload the sketch to the board;
5. configure the Arduino IoT Cloud dashboard widgets;
6. visualize the temperature and humidity values in real time.

Before uploading the sketch, create a local `arduino_secrets.h` file containing your own Wi-Fi credentials.

Do not commit real Wi-Fi credentials, tokens or private Arduino Cloud secrets to the repository.

## Role in the research workflow

This prototype represents the first step of the IoT monitoring workflow developed in the research.

It is used to test the basic acquisition and visualization of environmental data before moving to:

- proprietary streaming workflows;
- MQTT-based open-source workflows;
- database-driven visualization;
- BIM/openBIM and HBIM model-based visualization.
